display

  • display:block;
    • Voor elementen die een width, height, padding of margin nodig hebben maar standaard 'inline' zijn (a, label ...)
    • Heeft standaard 'width:auto;'
  • display:inline;
    • Voor elementen binnen een tekstregel
    • Gebruik je zelden omdat die elementen (strong, a, em, span) standaard al inline zijn
  • display:none;
    • Verberg een element
  • Vermijd andere display-values (inline-block ...). De ondersteuning door oudere browsers is niet erg betrouwbaar
  • Meer info vanaf slide 8 in les 2

height

  • In het ideale geval zorgt de inhoud van een block-element zélf voor de hoogte van dat element
  • Je gebruikt height dus énkel wanneer dat dat niet het geval is
    • Bv. Blokjes die naast elkaar staan en die dezelfde hoogte moeten hebben, maar een variabele inhoud hebben
    • height:100%; voor een element dat over de hele pagina moet staan.
      • Een procentuele height werkt énkel wanneer het parent-element ook een ingestelde height heeft
      • Je moet hier dus ook body en html height 100% geven.
  • Meer info vanaf slide 2 in les 4

width

  • Een block-element heeft standaard 'width:auto;'
    • inclusief borders, paddings en margins wordt de volledige breedte van het parent-element wordt ingenomen
  • Let op: Wanneer je een block-element 'width:100%' geeft
    • wordt exclusief borders, paddings en margins, de volledige breedte van het parent-element ingenomen
    • Het element komt dan dus buiten het parent-element
  • Meer info vanaf slide 2 in les 4

max-width, min-width, max-height ...

  • Werken maar wanneer de width of height variabel is
    • bv. wanneer de width of height opgegeven is in procenten
  • Vooral te gebruiken om layoutprobleempjes als te lange tekstregels op te lossen
  • Meer info vanaf slide 5 in les 6

padding

  • Creëer (een beetje) afstand tussen de inhoud van een element en de border
  • Gebruik dit niet om elementen te positioneren in een paginastructuur
    • Padding's van +500px = knoeiwerk
  • Meer info vanaf slide 5 in les 4

margin

  • Creëer (een beetje) afstand tussen de borders van twee elementen
  • Gebruik dit niet om elementen te positioneren in een paginastructuur
    • Margin's van +500px = knoeiwerk
  • Margin:auto;
    • Werkt enkel wanneer het element een opgegeven width heeft
    • Zorgt ervoor dat het element horizontaal in het midden van de parent staat
  • Tip: In sommige gevallen lopen margins buiten het parent element. Padding gebruiken lost dat probleem meestal op
  • Meer info vanaf slide 6 in les 4

float/clear

  • Positioneer block-elementen naast elkaar
    • Of laat tekst rond een element lopen
  • De beste, meest flexibele manier om pagina's te layouten
  • Een veel voorkomend 'probleem' is dat een parent-element floatende elementen niet lijkt te zien. Mogelijke oplossingen
    • overflow:auto; op het parent element
    • het parent element zelf laten floaten
    • 'clearfix' op het parent-element
  • Meer info vanaf slide 9 in les 4

position

  • Te gebruiken wanneer je elementen boven/onder elkaar wil zetten
  • Kan ook gebruikt worden voor hele paginastructuren maar gebruik dan liever float
  • Meer info in les 5

tables

  • Gebruik de table-tag énkel om gegevens/data in een tabel weer te geven
  • Daarvoor dient de table-tag

afbeeldingen

  • img-tag: afbeeldingen die op zichzelf, bijvoorbeeld in een tekst staan
  • background-image: afbeeldingen die achter een tekst of andere elementen staan
  • Maar!
    • Soms kan het interessanter zijn om een div met een background-image te gebruiken dan een img-tag ...
  • Supertip: Geef je een afbeelding enkel een width (middels css, of als html-attribuut) dan schaalt de height proportioneel mee.

nesting

  • Wees zuinig met het aantal html-niveau's, maar niet té zuinig. Soms is een extra niveau nodig of gewoon gemakkelijker
  • Aarzel niet om je ontwerp eerst onder te verdelen in enkele hoofdblokken en dan telkens op te splitsen per onderdeling
